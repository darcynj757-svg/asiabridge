**Ты — senior full-stack разработчик. Создай полностью рабочий веб-сервис B2B-платформы под названием AsiaBridge B2B.**

### Описание проекта
AsiaBridge B2B — это маркетплейс, который соединяет:
- Поставщиков (экспортёров) из России и других стран
- Импортёров из Камбоджи, Таиланда, Малайзии, Индонезии и других стран ЮВА

Платформа двусторонняя: и российские компании могут продавать в Азию, и азиатские — в Россию/другие страны.

### Технологический стек (обязательно использовать именно его):
- Python + Flask (последняя версия)
- Flask-Login + Flask-WTF + Flask-SQLAlchemy
- SQLite (база данных)
- Bootstrap 5 + custom CSS (современный чистый дизайн)
- Jinja2 шаблоны
- Всё в одном Replit-проекте (структура папок должна быть понятной)

### Структура проекта (обязательная):
```
/app
```

### Основные роли пользователей
1. **Supplier** (Поставщик / Экспортёр)
2. **Buyer** (Импортёр / Закупщик)
3. **Guest** (только просмотр)

### Функционал Личного Кабинета (обязательно для обеих ролей)

**Общий для всех:**
- Регистрация / Логин / Выход
- При регистрации пользователь сразу выбирает роль (Supplier / Buyer) и страну
- Профиль компании (название, логотип, описание, контакты, страна, категория бизнеса)

**Для Supplier (Поставщик):**
- Размещение товара (фото до 5 шт., название, описание, цена, MOQ, единица измерения, категория, страна происхождения, сертификаты)
- Управление своими товарами (редактировать, скрывать, удалять)
- Просмотр входящих запросов (RFQ)
- Чат с потенциальными покупателями
- Раздел «Мои сделки» со статусами

**Для Buyer (Импортёр):**
- Поиск и фильтры товаров (по категории, стране, цене, ключевым словам)
- Создание RFQ (запрос коммерческого предложения)
- Избранное товаров и поставщиков
- Чат с поставщиками
- Раздел «Мои запросы» и «Мои сделки»

### Главные страницы
1. Главная (landing) — красивый hero-блок + преимущества + кнопки «Стать поставщиком» и «Стать импортёром»
2. Каталог товаров (с фильтрами и поиском)
3. Страница товара (детальная)
4. Личный кабинет (дашборд с вкладками)
5. Чат (один-на-один между Supplier и Buyer)
6. Страница профиля компании
7. Админ-панель (простая, доступ только по роли admin — для модерации товаров и пользователей)

### Дополнительно (должно быть реализовано):
- Уведомления (внутри сайта — колокольчик)
- Статусы сделок: Новый запрос → Получено предложение → Переговоры → Договор → Отгрузка
- Мультиязычность (русский + английский) — переключатель в шапке
- Адаптивный дизайн (мобильные устройства)
- Защита от спама (CSRF, rate limit на регистрацию)
